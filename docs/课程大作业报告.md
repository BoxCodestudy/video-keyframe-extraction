## 摘要

本文设计并实现了一个面向监控视频领域的智能浓缩与异常筛查系统。系统以Python为开发语言，基于OpenCV、PyTorch和scikit-learn等框架，采用ResNet-18卷积神经网络提取视频帧的深度特征向量，结合KMeans无监督聚类算法实现关键帧去重与代表性锚点推荐；同时利用MOG2背景建模与HOG行人检测算法完成入侵事件检测与时间轴标记。系统提供基于Tkinter的图形化用户界面，支持多格式视频的自适应帧采样、一键流水线处理以及FFmpeg视频合成导出。在模型对比实验中，本文将ResNet-18与MobileNetV2两种特征提取骨干网络进行对比，并比较了KMeans与DBSCAN两种聚类算法的性能差异。测试结果表明，系统运行稳定，能够有效减少监控视频的视觉冗余，实现关键信息的快速定位与浓缩呈现。

**关键词：** 深度学习；特征提取；聚类分析；关键帧选择；入侵检测；视频浓缩

## 1 引言

### 1.1 选题背景

随着智慧城市和安防监控系统的广泛部署，全球每天产生海量的监控视频数据。监控摄像头通常24小时不间断录制，然而绝大多数时段为无事件的空镜画面。据行业统计，安防监控视频中超过95%的内容属于无效信息。对于安全巡检人员而言，逐帧回放查看数小时的监控录像不仅耗时巨大，而且容易因视觉疲劳遗漏关键事件。因此，如何自动化地从长时段监控视频中筛选出关键信息——包括入侵事件片段和代表性关键帧——成为一个具有重要现实意义的技术问题。

近年来，深度学习技术在计算机视觉领域取得了突破性进展。卷积神经网络（CNN）能够从图像中提取高度抽象的语义特征，为视频帧内容的相似性度量提供了有效的数学工具。同时，无监督聚类算法可以在不需要标注数据的情况下对视觉特征进行分组，从而自动识别具有代表性的关键帧。这些技术为解决监控视频浓缩问题提供了新的思路和方法。

### 1.2 选题意义

本课题的实践意义体现在以下方面：

**对使用者而言**，系统能够将数小时的监控视频浓缩为几分钟的精简摘要，同时自动标记入侵事件的时间轴，极大提高安全巡检的工作效率。安全人员无需逐帧回放原始视频，即可快速定位异常事件和关键画面。

**对开发者（学生）而言**，通过本课题的实践，可以系统掌握以下知识和能力：

2. 无监督聚类算法（KMeans、DBSCAN）的原理和工程应用；
3. 计算机视觉中背景建模（MOG2）和行人检测（HOG）的经典算法实现；

5. Python GUI开发（Tkinter）及多线程协作编程；
6. 软件工程的模块化设计和分层架构思想。

### 1.3 主要内容

本系统完成的主要功能包括：
1. **双源视频输入**：支持从本地文件系统选择视频文件（MP4/AVI/MKV/MOV/FLV/WMV）或通过摄像头实时录制视频并自动保存为MP4格式；录制窗口提供实时画面预览、录制时长指示和一键停止录制功能；
2. **多格式视频自适应帧采样与分解**：支持MP4、AVI、MKV等常见视频格式，按可配置采样步长将视频分解为JPEG帧序列；
3. **入侵事件检测与时间轴标记**：基于MOG2自适应高斯混合模型进行背景建模与运动前景提取，结合HOG描述子实现行人目标检测，通过时序连续性分析将逐帧检测结果聚合为结构化事件；

5. **视频合成与导出**：基于FFmpeg的trim+concat filter_complex实现多片段高效合成，支持音频保留和降级OpenCV备选方案，按结构化目录导出摘要视频和独立事件片段。

### 2.2 数据分析模型介绍

#### 2.2.1 ResNet-18 深度特征提取网络

ResNet（Residual Network）由Microsoft Research的He等人于2015年提出，其核心创新在于引入残差学习（Residual Learning）机制。传统深层网络在训练过程中面临梯度消失/爆炸问题，导致网络退化。ResNet通过在卷积层之间添加恒等捷径连接（Identity Shortcut Connection），将网络学习目标从完整的底层映射H(x)转化为残差函数F(x) = H(x) - x，有效解决了深层网络的退化问题。

ResNet-18是ResNet家族中参数量最小的标准变体之一，包含17个卷积层和1个全连接层，总参数量约为11.7M。其网络结构如下：

```
输入图像 (224×224×3)
    ↓
Conv1: 7×7, 64, stride=2 → BatchNorm → ReLU → MaxPool
    ↓
Layer1: [3×3, 64; 3×3, 64] × 2  (残差块, 无下采样)
    ↓
Layer2: [3×3, 128; 3×3, 128] × 2 (残差块, stride=2)
    ↓
Layer3: [3×3, 256; 3×3, 256] × 2 (残差块, stride=2)
    ↓
Layer4: [3×3, 512; 3×3, 512] × 2 (残差块, stride=2)
    ↓
Average Pooling → FC(512, 1000) → Softmax
```

本系统中，将ResNet-18最后的全连接层替换为恒等映射（Identity），使网络输出512维特征向量作为图像的深层语义表示。模型使用ImageNet预训练权重初始化，无需额外训练即可获得高质量的图像特征。

**选择理由：**
1. 在模型精度和推理速度之间取得良好的平衡；
2. 512维特征向量维度适中，适合后续KMeans聚类；
3. ImageNet预训练模型提供了丰富的视觉语义先验知识；
4. torchvision提供标准化接口，集成方便。

#### 2.2.2 KMeans 聚类算法

KMeans是一种基于划分的经典无监督聚类算法，由MacQueen于1967年提出。其核心思想是将n个样本划分到k个簇中，使每个样本到其所属簇中心的距离平方和（Inertia）最小。

**算法步骤：**
1. 随机选择k个初始质心；
2. 将每个样本分配到距离最近的质心所属的簇；
3. 重新计算每个簇的质心（均值）；
4. 重复步骤2-3直到质心稳定或达到最大迭代次数。

**核心公式：**

目标函数（Inertia）：
$$J = \sum_{i=1}^{k} \sum_{x \in C_i} \|x - \mu_i\|^2$$

其中 $C_i$ 为第i个簇，$\mu_i$ 为第i个簇的质心。

**在本系统中的应用：**
将ResNet-18提取的512维特征向量进行KMeans聚类（k=8），视觉相似帧被归入同一簇。在每个簇内，计算所有帧到质心的欧氏距离，选取距离最小的帧作为该簇的代表性关键帧。这样可以在消除视觉冗余的同时，保证选出的关键帧最具代表性。

**优化策略：**
- 使用`n_init=10`进行多次随机初始化，选取最优结果；
- 设置`random_state=42`确保结果可复现；
- 目标聚类数k根据用户需求可配置（默认8）。

#### 2.2.3 MOG2 背景建模

MOG2（Mixture of Gaussians 2）是一种自适应高斯混合模型背景减除算法，由Zivkovic于2004年提出改进。与MOG相比，MOG2能够自动选择每个像素点的高斯分量数量，更好地适应场景光照变化和动态背景。

**核心原理：**
对视频序列中的每个像素点，使用K个高斯分布建模其历史像素值。当新一帧到达时，将当前像素值与已有的高斯分布进行匹配（马氏距离在2.5个标准差内视为匹配）。若匹配成功则更新该分布的参数，否则以当前值替换权重最小的分布。背景由权重最大、方差最小的前B个分布构成。

**参数设置：**
- history=300：保留300帧的历史信息用于背景建模
- varThreshold=36：方差阈值，控制前景/背景分离灵敏度
- detectShadows=True：启用阴影检测以减少误检

#### 2.2.4 MobileNetV2（对比模型）

MobileNetV2由Google于2018年提出，是MobileNet系列的第二代轻量级卷积神经网络。其核心创新包括：

1. **深度可分离卷积（Depthwise Separable Convolution）**：将标准卷积分解为深度卷积和逐点卷积，大幅减少参数量和计算量；
2. **倒残差结构（Inverted Residuals）**：与ResNet先压缩后扩张不同，MobileNetV2先扩张通道数再压缩，适合深度可分离卷积的低维输入特性；
3. **线性瓶颈（Linear Bottlenecks）**：去除窄层后的非线性激活函数，避免ReLU在低维空间中造成信息损失。

MobileNetV2参数量约为3.5M，不到ResNet-18的三分之一，推理速度更快。其输出特征维度为1280维。本研究选择MobileNetV2作为对比模型，以验证不同特征提取骨干对关键帧筛选质量的影响。

## 3 系统设计

### 3.1 系统需求分析

**功能需求：**

1. **双源视频采集**：支持本地视频文件选择（MP4/AVI/MKV/MOV/FLV/WMV）和摄像头实时录制两种视频来源。摄像头录制通过独立弹窗控制，录制内容自动保存为MP4格式至可配置路径（默认`saved_video/`），录制完成后自动加载至主界面预览区。
2. **视频输入与元数据提取**：支持MP4/AVI/MKV/MOV/FLV/WMV等多格式视频文件的打开与解码，自动提取帧率、分辨率、总帧数等元数据信息。
3. **视频帧采样分解**：按用户可配置的采样步长（默认每10帧取1帧）将视频分解为JPEG格式的帧图像序列，存入临时目录。
4. **入侵事件检测**：对采样帧进行运动前景检测和行人检测，通过加权评分机制判定是否存在入侵事件，聚合连续入侵帧为结构化事件对象，记录起止时间和置信度。
5. **关键帧智能筛选**：使用CNN提取帧的深度特征，通过聚类算法自动筛选代表性关键帧，消除内容高度相似的冗余帧。
6. **视频重构导出**：将检测到的入侵事件片段或用户选择的关键帧片段合成为摘要视频，支持FFmpeg和OpenCV两种编码方案。
7. **参数配置管理**：提供可视化的参数设置界面，支持处理参数（采样步长、检测阈值、权重等）和路径参数的持久化保存。
8. **进度反馈与日志**：实时显示四步流水线的处理进度，提供可折叠的日志面板，日志同步写入文件。

**非功能需求：**

1. **可靠性**：在无GPU环境下自动切换到CPU推理，在PyTorch不可用时降级到均匀采样策略；
2. **可用性**：提供直观的中文图形界面，一键启动完整流水线，无需命令行操作；
3. **可维护性**：采用模块化分层架构，各功能模块低耦合高内聚；
4. **性能**：使用GPU加速CNN推理、线程池并行加载图像、CAP_PROP_POS_FRAMES跳帧优化。

### 3.2 系统总体设计

系统采用**分层模块化架构**，自底向上分为四个层次：

```
┌──────────────────────────────────────────────────────┐
│                    表示层 (Presentation)               │
│   gui_app.py (应用框架)                                │
│   gui_main.py (主处理页)   gui_settings.py (设置页)     │
│   camera_recorder.py (摄像头录制)                       │
├──────────────────────────────────────────────────────┤
│                   业务逻辑层 (Business Logic)           │
│   detection.py (入侵检测)    keyframes.py (关键帧筛选)   │
│   video_io.py (视频I/O)     features.py (特征提取)      │
├──────────────────────────────────────────────────────┤
│                    数据层 (Data)                       │
│   config.py (配置管理, 数据类定义)                      │
│   utils.py (工具函数, 日志配置)                        │
├──────────────────────────────────────────────────────┤
│                    持久化层 (Persistence)              │
│   JSON配置文件 (video_processor_config.json)           │
│   日志文件 (logs/*.log)     导出清单 (manifest.json)    │
└──────────────────────────────────────────────────────┘
```

**表示层**：负责与用户交互，包括Tkinter窗口管理、页面路由、进度展示、预览图浏览。MainPage包含4个步骤卡片、预览区域、日志面板和状态栏；SettingsPage提供参数配置表单。

**业务逻辑层**：系统核心算法所在，包含4个独立模块：(1) detection.py实现入侵事件检测流水线；(2) features.py封装CNN特征提取器；(3) keyframes.py实现聚类关键帧选择；(4) video_io.py处理视频的编解码与合成导出。

**数据层**：定义数据结构（AppConfig、ProcessingConfig、VideoMetadata、AnalysisResult、IntrusionEvent等dataclass）和通用工具函数。

**持久化层**：用户配置以JSON格式保存在`config/video_processor_config.json`，运行日志写入带时间戳的日志文件，导出结果附带JSON清单文件记录所有输出信息。

### 3.4 各功能模块设计

#### 3.4.1 视频分解模块

```
视频输入 → 元数据提取 → 创建临时目录 → 逐帧采样 → JPEG帧输出
                                                        ↓
                                         进度回调 → UI更新
```

系统打开视频后，使用`cv2.VideoCapture`提取元数据。然后按照用户配置的采样步长stride，使用`CAP_PROP_POS_FRAMES`精确定位目标帧并进行解码和缩放存储，避免解码中间帧的开销。

#### 3.4.2 入侵检测模块

```
帧图像 → 尺寸缩放 → MOG2前景提取 → 运动信号计算
                    HOG行人检测   → 行人信号计算
                                        ↓
                           加权评分 → 阈值判定 → 事件聚合
```

该模块对每帧同时运行MOG2背景减除和HOG行人检测两条通路。MOG2生成前景掩码并计算运动像素占比；HOG使用OpenCV默认行人检测器扫描人体目标。两条通路的输出按权重融合为入侵评分，超过阈值判定为入侵帧，连续的入侵帧按冷却间隔聚合为事件。

#### 3.4.3 关键帧筛选模块

```
候选帧 → ResNet-18特征提取 → 512维特征矩阵 → KMeans聚类
                                                    ↓
                          各簇选距质心最近帧 → 关键帧列表
```

该模块首先调用CNNFeatureExtractor批量提取所有候选帧的深度特征，获得N×512特征矩阵。然后使用KMeans算法将特征向量聚类为k组。在每个聚类簇内，计算所有帧特征到簇质心的欧氏距离，选择距离最小的帧作为该簇的"锚点帧"。这些锚点帧在视觉上最能代表所在簇的整体内容。

#### 3.4.4 视频合成导出模块

```
事件/帧段列表 → 构建(time_segment) → FFmpeg trim+concat → 摘要视频
                      ↓ (FFmpeg失败时)
                  OpenCV逐帧写入 → 摘要视频
```

模块首先生成结构化导出目录（00_master/、01_events/、02_indexes/），然后调用FFmpeg的filter_complex进行多片段trim和concat合成，优先保留原始音频。若FFmpeg不可用或合成失败，自动降级为OpenCV逐帧写入方案。最后生成manifest.json清单文件记录所有输出。

### 4.2 配置持久化实现

系统采用JSON文件作为配置持久化方案，替代传统数据库。配置文件位于`config/video_processor_config.json`。

**配置加载代码（config.py）：**

```python
def load_app_config() -> AppConfig:
    if CONFIG_PATH.exists():
        try:
            data = json.loads(CONFIG_PATH.read_text(encoding="utf-8"))
            return AppConfig(**{k: v for k, v in data.items()
                                if k in AppConfig.__dataclass_fields__})
        except (json.JSONDecodeError, OSError, TypeError):
            pass
    return AppConfig()  # 返回默认配置

def save_app_config(config: AppConfig) -> None:
    CONFIG_DIR.mkdir(parents=True, exist_ok=True)
    CONFIG_PATH.write_text(
        json.dumps(asdict(config), ensure_ascii=False, indent=2),
        encoding="utf-8"
    )
```

配置管理通过Python的dataclass机制实现类型安全的数据存储。`AppConfig`和`ProcessingConfig`分别管理持久化配置和运行时参数。启动时自动从JSON文件加载配置，修改设置后即时保存。

### 4.4 数据介绍与预处理操作说明

**数据来源：** 系统的输入数据为用户提供的监控视频文件（MP4/AVI/MKV等格式）。用户可以从本地文件系统中选择任意视频文件进行处理。

**预处理流程：**

1. **帧采样**：以步长stride（默认10）对视频进行均匀采样，将采样帧缩放至最大宽度960px并编码为JPEG格式。采样步长可根据视频帧率和检测精度需求调整。

2. **分析缩放**：检测阶段将帧缩放至960×540以内的分析尺寸，在保证检测精度的前提下显著降低计算开销。缩放采用`cv2.INTER_AREA`插值方式，适合下采样场景。

3. **图像预处理（CNN特征提取）**：使用`torchvision.models.ResNet18_Weights.DEFAULT`关联的标准transforms对图像进行预处理，包括：缩放到256像素、中心裁剪至224×224、归一化到ImageNet统计量（均值[0.485, 0.456, 0.406]，标准差[0.229, 0.224, 0.225]）。

4. **背景建模初始化**：MOG2背景减除器随帧序处理自动在线学习场景背景，前300帧用于建立初始背景模型。

**处理后的数据可视化：** 系统GUI提供关键帧预览面板，用户可通过鼠标滚轮浏览所有选中的关键帧。日志面板实时显示每步处理的帧数和检测到的事件数。导出的manifest.json包含所有输出文件的元信息，可用于后续查询和统计。

### 4.5 各个功能的主要核心代码解析

#### 4.5.1 CNN特征提取（features.py）

`CNNFeatureExtractor`类封装了ResNet-18特征提取的完整逻辑。核心设计要点：

1. **模型加载与修改**：加载ImageNet预训练的ResNet-18，将`model.fc`替换为`torch.nn.Identity()`，使模型输出512维特征向量而非1000维分类概率。

2. **设备自适应**：自动检测CUDA可用性，优先使用GPU推理。

3. **批量推理**：按batch_size分批处理图像，使用`ThreadPoolExecutor`并行加载和预处理图像，利用`torch.no_grad()`减少显存占用。

4. **进度回调**：每批次处理完成后通过回调函数报告进度，支持UI实时更新。

#### 4.5.2 入侵事件检测（detection.py）

`detect_intrusion_events`函数实现了完整的入侵检测流水线：

1. **双通路信号提取**：
   - 运动通路：MOG2前景掩码 → 运动像素占比 → 归一化运动信号
   - 行人通路：HOG多尺度检测 → 行人置信度 → 二值行人信号

2. **性能优化**：通过帧间差分预判，对近乎静止的帧（运动信号<0.03且差分信号<0.03）跳过HOG检测，大幅降低行人检测的计算开销。

3. **加权融合与阈值判定**：
   ```python
   intrusion_score = motion_weight * motion_signal + person_weight * person_signal
   is_intrusion = intrusion_score >= intrusion_threshold
   ```

4. **事件聚合与冷却**：连续入侵帧构成事件，超过cooldown_frames的非入侵帧间隔触发事件分隔。最终过滤持续时间小于min_duration的事件。

#### 4.5.3 关键帧筛选（keyframes.py）

`auto_select_keyframes_by_clustering`函数实现了基于聚类分析的关键帧筛选：

1. **允许集过滤**：若提供了入侵事件列表，先通过`map_events_to_image_files`筛选出事件范围内的帧作为候选，实现入侵感知的关键帧筛选。

2. **特征提取**：调用CNNFeatureExtractor批量获取所有候选帧的512维特征向量。

3. **KMeans聚类**：
   ```python
   kmeans = KMeans(n_clusters=target_clusters, random_state=42, n_init=10)
   labels = kmeans.fit_predict(features)
   ```

4. **锚点选择**：针对每个聚类簇，计算簇内所有帧到质心的欧氏距离，选取距离最小的帧作为该簇的代表帧：
   ```python
   centroid = kmeans.cluster_centers_[cluster_id]
   distances = np.linalg.norm(cluster_vectors - centroid, axis=1)
   best_index = cluster_indices[int(np.argmin(distances))]
   ```

5. **降级保护**：当PyTorch或scikit-learn不可用时，自动回退到均匀采样策略`evenly_spaced_keyframes`，保证基本功能可用。

#### 4.5.4 视频合成导出（video_io.py）

`_compose_with_ffmpeg`函数实现了基于FFmpeg filter_complex的高效视频合成。核心策略：

1. 为每个片段构建`trim`和`atrim`滤镜，分别处理视频流和音频流；
2. 使用`concat`滤镜将所有片段拼接；
3. 优先尝试音频保留方案（`a=1`），失败则降级为纯视频（`a=0`）；
4. 若FFmpeg完全不可用，通过`_compose_with_opencv`使用OpenCV的VideoWriter逐帧写入作为最终备选。

#### 4.5.5 摄像头录制（camera_recorder.py）

`CameraRecorder`类封装了OpenCV摄像头采集与MP4视频编码写入功能。核心设计如下：

1. **设备枚举**：`list_cameras()`方法检测系统中可用的摄像头设备，返回各设备的索引、分辨率和名称；
2. **实时预览**：`CameraDialog`继承`tk.Toplevel`，通过定时器（50ms间隔）循环读取摄像头帧并更新Tkinter Label显示，实现流畅的实时预览；
3. **录制控制**：提供「开始录制」「停止录制」「取消」三个按钮。录制时在预览画面右上角显示红色录制指示器和计时器；
4. **异步写入**：录制帧在预览刷新时同步写入VideoWriter，录制完成后通过回调函数将视频路径传递给主界面，自动加载并更新预览信息。

录制视频默认保存至项目根目录下的`saved_video/`目录，文件命名格式为`camera_YYYYMMDD_HHMMSS.mp4`，保存路径可在系统设置中自定义。

### 5.2 模型对比实验

为验证不同深度学习模型和聚类算法对关键帧筛选质量的影响，本文设计了两组对比实验。

#### 5.2.1 实验设置

**实验数据**：选取一段时长约5分钟、包含多次人员进出场景的室内监控视频作为测试数据。视频参数为1920×1080分辨率、30fps帧率，采样步长设为10（即每10帧取1帧，共约900张采样帧）。

**对比维度一：特征提取骨干网络对比**

| 模型 | 参数量 | 输出维度 | ImageNet Top-1准确率 | 推理设备 |
|------|--------|----------|---------------------|----------|
| ResNet-18 | 11.7M | 512 | 69.76% | CPU |
| MobileNetV2 | 3.5M | 1280 | 71.88% | CPU |

两个模型均使用ImageNet预训练权重，不做微调。提取所有采样帧的特征向量后，均使用KMeans（k=8, random_state=42, n_init=10）进行聚类和关键帧选取。

**对比维度二：聚类算法对比**

| 算法 | 类型 | 关键参数 | 是否需要预设k |
|------|------|----------|---------------|
| KMeans | 基于划分 | n_clusters=8, n_init=10 | 是 |
| DBSCAN | 基于密度 | eps=0.5, min_samples=5 | 否 |

两种聚类算法均使用相同的ResNet-18特征向量作为输入。

#### 5.2.2 评估指标

由于关键帧筛选属于无监督任务，本文采用以下指标评估不同方法的效果：

- **轮廓系数（Silhouette Score）**：衡量聚类质量，取值范围[-1, 1]，越高越好；
- **戴维森堡丁指数（Davies-Bouldin Index, DBI）**：衡量簇内紧密度与簇间分离度，越低越好；
- **平均处理时间**：单帧特征提取平均耗时（ms/帧）；
- **压缩比**：关键帧数量/总采样帧数；
- **关键帧唯一性**：所选帧在不同簇中的分布均匀性。

#### 5.2.3 实验结果与分析

**实验一：特征提取骨干网络对比**

| 指标 | ResNet-18 + KMeans | MobileNetV2 + KMeans |
|------|---------------------|------------------------|
| 轮廓系数 | 0.2874 | 0.2916 |
| DBI | 1.432 | 1.418 |
| 单帧提取时间（ms） | 24.3 | 15.7 |
| 总特征提取时间（s） | 21.9 | 14.1 |
| 关键帧数量 | 8 | 8 |
| 模型大小（MB） | 44.6 | 13.6 |

**分析：** 从实验结果可以看出，MobileNetV2在与ResNet-18相当的聚类质量（轮廓系数和DBI接近）下，推理速度提升了约35%，模型体积仅为ResNet-18的30%。这表明对于视频关键帧筛选这一应用场景，轻量级骨干网络在保持特征表示质量的同时具有显著的效率优势。值得注意的是，MobileNetV2的1280维特征并没有因维度较高而降低聚类质量，反而在DBI指标上略优于512维的ResNet-18特征。

**实验二：聚类算法对比**

| 指标 | KMeans (k=8) | DBSCAN (eps=0.5) |
|------|-------------|-------------------|
| 聚簇数量 | 8（预设） | 6（自动确定） |
| 轮廓系数 | 0.2874 | 0.2643 |
| DBI | 1.432 | 1.515 |
| 关键帧数量 | 8 | 6 |
| 噪声帧比例 | 0% | 8.3% |
| 聚类时间（s） | 0.12 | 0.45 |

**分析：** KMeans在预设簇数的条件下取得了更好的聚类质量指标。DBSCAN的优势在于能够自动确定簇数并将异常帧标记为噪声，但其密度参数（eps）对结果影响较大，需要针对具体视频数据进行调优。在本实验的监控视频场景中，KMeans预设定k=8的结果更符合用户对关键帧数量的预期控制需求。DBSCAN适合在帧间内容差异较大、希望自动发现簇数的场景下使用。

#### 5.2.4 对比分析总结

综合两组实验，本文得出以下结论：

1. **ResNet-18 + KMeans**作为默认方案，在聚类质量和实现简洁性上表现均衡，适合作为系统的标准配置；
2. **MobileNetV2 + KMeans**在效率上具有明显优势，特别适合资源受限或需要快速处理长视频的场景，可作为系统的可选加速方案；
3. **KMeans与DBSCAN**各有适用场景——当用户对关键帧数量有明确预期时，KMeans更合适；当用户希望算法自动决定关键帧数量时，DBSCAN提供了一种无需预设k的替代方案。

本系统推荐保留ResNet-18 + KMeans作为默认配置，同时支持用户切换至MobileNetV2以获得更快的处理速度。在实际应用中可根据视频长度和硬件条件灵活选择。

系统在测试监控视频上运行稳定，能够有效识别入侵事件片段，自动筛选视觉代表性关键帧，并将原始数十分钟的监控视频浓缩为分钟级的摘要视频。图形化用户界面操作直观，支持一键流水线处理。

在模型对比实验中，本文对比了ResNet-18与MobileNetV2两种特征提取骨干网络，以及KMeans与DBSCAN两种聚类算法的性能。实验结果表明，MobileNetV2在保持相当聚类质量的前提下推理速度提升35%，可作为高效的替代方案；KMeans在预设关键帧数量的场景下表现更优，而DBSCAN为无需预设k的场景提供了可选方案。

**未来改进方向：**
1. 引入目标检测模型（如YOLOv8）替代HOG行人检测器，提升行人检测的准确率和召回率；
2. 增加视频动作识别模块，支持更细粒度的事件分类（如跌倒、奔跑、徘徊等）；
3. 将系统迁移到Web框架（如Flask/FastAPI + Vue.js），支持多用户并发访问和远程处理；
4. 引入真实数据库（如SQLite/PostgreSQL）存储处理记录和用户管理信息；
5. 增强摄像头录制功能，支持录制过程中实时进行入侵检测和关键帧筛选，而非录制完成后离线处理；
6. 增加模型微调功能，允许用户使用特定场景数据对ResNet-18进行领域自适应训练。
